# 1-Analisando estruturas de prompts (Resumo)

## 1) O que é

Estrutura básica de prompt para organizar instruções de forma mais clara. Permite criar prompts mais organizados e eficientes.

## 2) Para que serve

Organizar prompts de forma estruturada. Aplicável em diferentes contextos, não apenas desenvolvimento. Útil para integrar IA em empresas e passar instruções claras para agentes.

## 3) Como é usado (como escrever / o que incluir no prompt e no comportamento)

Usar a mesma estrutura de prompt para diferentes finalidades. Exemplo: verificar dependências de bibliotecas, checar versões, identificar bibliotecas depreciadas ou com falhas de segurança.

A estrutura inclui: persona, escopo, objetivo, entrada, saída, critério, ambiguidade, instruções negativas.

## 4) Onde pode ser usado (casos de uso)

- Verificação de vulnerabilidades em projetos
- Auditoria de dependências
- Integração de IA em empresas
- Criação de agentes com instruções claras

## 5) Limitações / quando evitar

Não é uma estrutura única que resolve todos os problemas. É um bom começo, mas pode precisar de adaptações conforme o contexto.

## 6) Observações de prática (coisas que dá pra notar quando testa no dia a dia)

A estrutura serve como base, mas cada caso pode precisar de ajustes. É importante testar e adaptar conforme necessário.
